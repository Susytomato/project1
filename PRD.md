[贪吃蛇游戏] PRD
一、專案概述
    專案名稱：高品质网页版贪吃蛇（Snake Game）
    一句話描述：基于 HTML5 Canvas + JavaScript 开发的高品质视觉风格贪吃蛇游戏，支持多趣味模式、分数统计、速度递增、暂停 / 继续等核心功能，具备响应式适配能力。
    版本：V2.0（新增障碍模式、双倍食物模式）
二、問題與目標
    問題陳述
    传统网页贪吃蛇视觉效果简陋，缺乏沉浸式体验
    部分贪吃蛇游戏存在按键防抖问题（快速切换方向导致蛇身反向）
    游戏难度缺乏梯度，后期趣味性不足
    缺乏完整的状态反馈（暂停、结束、速度提升）和响应式适配，移动设备体验较差
    单一游戏模式可玩性有限，难以满足不同用户的游戏偏好（休闲 / 挑战 / 策略）
    目標
    实现一款视觉高品质、具备立体感和发光效果的贪吃蛇游戏
    解决按键防抖问题，优化操作流畅度
    加入速度递增机制，构建梯度化游戏难度
    完善游戏状态反馈，支持暂停 / 继续、重新开始功能
    实现响应式适配，支持平板、小屏电脑等设备正常显示
    新增多趣味游戏模式，满足不同用户偏好，提升长期可玩性：
    障碍模式：增加随机固定障碍，提升游戏挑战性
    双倍食物模式：增加高分策略性，满足追求高分的用户需求
    实现模式切换功能（游戏未运行时可自由切换），界面清晰展示当前模式
三、目標用戶
    用戶特徵
    喜欢休闲益智类网页小游戏的互联网用户
    对游戏视觉效果有一定要求，追求基础沉浸式体验
    无需安装额外软件，希望通过浏览器直接打开游玩
    年龄跨度广，操作门槛低，适合碎片化时间消遣
    存在不同游戏偏好：
    休闲型：喜欢无干扰的基础玩法，放松心情
    挑战型：喜欢有障碍的高难度玩法，追求突破自我
    策略型：喜欢高分玩法，通过规划路线获取更高分数
    用戶需求
    操作简单：支持方向键控制蛇身移动，空格控制暂停 / 继续
    反馈清晰：实时查看当前分数、游戏模式、游戏状态（准备 / 进行 / 暂停 / 结束）
    难度合理：游戏后期逐步提升速度，增加挑战性但不影响操作
    视觉舒适：界面美观，色彩搭配协调，无刺眼元素
    便捷重启：游戏结束后可快速重新开始，无需刷新页面
    模式可选：支持根据自身偏好选择不同游戏模式，游戏未运行时可自由切换
    模式差异化：不同模式具备独特的游戏规则和视觉标识，玩法区分清晰
四、用戶故事
    用戶故事 1：作为一名休闲游戏爱好者，我想要通过浏览器直接打开贪吃蛇游戏，选择基础模式游玩，以便在碎片化时间进行消遣、放松心情。
    用戶故事 2：作为一名追求视觉体验的用户，我想要一款视觉效果精致的贪吃蛇游戏，以便获得更好的游玩沉浸感。
    用戶故事 3：作为一名喜欢有挑战梯度的用户，我想要游戏后期逐步提升速度，同时选择障碍模式，以便增加游戏挑战性，突破自我记录。
    用戶故事 4：作为一名偶尔需要中断游玩的用户，我想要游戏支持暂停 / 继续功能，以便在有其他事务时临时中断游戏，后续恢复游玩。
    用戶故事 5：作为一名追求高分的策略型用户，我想要双倍食物模式，以便通过规划最优路线获取更高分数，满足成就感。
    用戶故事 6：作为一名灵活调整玩法的用户，我想要在游戏未开始时自由切换不同模式，以便根据当下心情选择合适的游玩方式。
五、功能需求與驗收標準
    功能 1：[游戏核心操作（蛇身移动）]
    描述：支持上下左右方向键控制蛇身移动，蛇身会沿指定方向持续前进，禁止蛇身直接反向移动（按键防抖）。
    驗收標準（Given/When/Then）：
    Given 游戏处于运行状态，蛇身当前方向为 “向右”
    When 用户按下 “左方向键”
    Then 蛇身不执行反向移动，仍保持向右前进
    When 用户按下 “上方向键” 或 “下方向键”
    Then 蛇身成功切换为对应方向，流畅前进，无卡顿
    功能 2：[分数统计与食物生成]
    描述：蛇身吃到食物后，按当前模式获得对应分数，蛇身加长一节，同时在非蛇身 / 障碍区域生成新食物（双倍模式生成 2 个食物）。
    驗收標準（Given/When/Then）：
    Given 游戏处于运行状态，当前模式为基础 / 障碍模式，蛇头与食物坐标重合
    When 游戏完成一次帧循环
    Then 分数增加 10 分并伴随轻微缩放动画，蛇身加长一节，新食物在非蛇身 / 障碍区域生成，无重叠情况
    Given 游戏处于运行状态，当前模式为双倍食物模式，蛇头与任意一个食物坐标重合
    When 游戏完成一次帧循环
    Then 分数增加 15 分并伴随轻微缩放动画，蛇身加长一节，两个新食物在非蛇身 / 非彼此区域生成，无重叠情况
    功能 3：[速度递增机制]
    描述：每吃掉指定数量食物（5 个），游戏速度提升 10%，速度提升有上限，不超过最小移动间隔。
    驗收標準（Given/When/Then）：
    Given 游戏处于运行状态，已吃掉 5 个食物（基础 / 障碍模式累计 50 分，双倍食物模式累计 75 分）
    When 蛇身吃掉第 5 个食物完成帧循环
    Then 游戏速度提升 10%，界面显示 “速度提升！” 提示，2 秒后恢复默认状态文字，速度不低于 60ms / 步的上限
    功能 4：[游戏状态控制（暂停 / 继续 / 结束 / 重启）]
    描述：支持空格键暂停 / 继续游戏，游戏结束后禁用操作，点击 “开始 / 重新开始” 按钮可重置所有状态并启动游戏。
    驗收標準（Given/When/Then）：
    Given 游戏处于运行状态
    When 用户按下空格键
    Then 游戏暂停，定时器停止，界面显示 “游戏暂停”，再次按下空格键恢复运行，界面显示 “游戏进行中”
    When 蛇头撞墙、咬到自身或撞到障碍（障碍模式）
    Then 游戏结束，定时器停止，界面显示 “游戏结束！”，方向键和空格键操作失效
    When 用户点击 “开始 / 重新开始” 按钮
    Then 所有游戏状态（分数、蛇身、速度、方向、障碍 / 食物）重置，游戏重新启动，界面恢复正常操作状态
    功能 5：[多趣味模式切换与运行]
    描述：支持 3 种游戏模式（基础 / 障碍 / 双倍食物），游戏未运行时可自由切换，界面展示当前激活模式，不同模式具备独特规则和视觉元素。
    驗收標準（Given/When/Then）：
    Given 游戏处于未运行状态（准备开始 / 游戏结束）
    When 用户点击 “障碍模式” 按钮
    Then 按钮变为激活样式，界面显示当前模式为 “障碍模式”，重新开始游戏后画布生成 10 个灰色障碍，蛇撞到障碍直接结束
    When 用户点击 “双倍食物” 按钮
    Then 按钮变为激活样式，界面显示当前模式为 “双倍食物”，重新开始游戏后画布生成 2 个不同颜色食物，吃掉任意一个得 15 分
    When 用户点击 “基础模式” 按钮
    Then 按钮变为激活样式，界面显示当前模式为 “基础模式”，重新开始游戏后画布无障碍、仅生成 1 个食物，吃掉得 10 分
    Given 游戏处于运行状态
    When 用户点击任意模式按钮
    Then 模式无切换，保持当前运行模式不变，按钮无状态变化
    功能 6：[响应式适配]
    描述：支持平板、小屏电脑等设备，屏幕宽度≤900px 时，游戏容器改为纵向排列，优化显示效果。
    驗收標準（Given/When/Then）：
    Given 用户在平板设备（屏幕宽度≤900px）上打开游戏
    When 页面加载完成或窗口尺寸调整至≤900px
    Then 游戏容器从横向排列改为纵向排列，标题、游戏画布、信息面板依次垂直展示，无布局错乱，所有功能（模式切换、游戏运行）正常可用
六、技術約束
    必須遵守
    开发技术：基于 HTML5 Canvas 进行游戏渲染，原生 JavaScript 实现核心逻辑，不依赖第三方框架 / 库
    代码规范：代码结构清晰，分模块注释，变量 / 函数命名语义化，便于后续维护
    视觉标准：严格按照预设颜色配置实现视觉效果，保留网格线、圆角、发光阴影等高品质视觉元素，不同模式的元素（障碍、第二个食物）具备独特视觉标识
    性能要求：游戏帧循环无明显卡顿，食物 / 障碍生成算法高效，无无限循环卡顿风险
    模式规则：不同模式的核心规则（分数、障碍、食物数量）严格区分，运行时不互相干扰，模式切换仅在游戏未运行时生效
    兼容性要求
    浏览器兼容：支持现代主流浏览器（Chrome、Edge、Firefox、Safari）最新版本
    设备兼容：支持桌面端、平板设备，屏幕分辨率不低于 768px×600px
    交互兼容：支持键盘方向键、空格键，支持鼠标点击按钮（模式切换、重启）操作
    不要做
    不使用第三方 JavaScript 框架（如 jQuery、Vue、React），保持原生实现
    不添加复杂音效、广告等额外元素，聚焦游戏核心功能和趣味模式
    不支持移动端触屏操作（本版本仅聚焦键盘操作和桌面 / 平板端）
    不保存游戏进度和历史最高分，仅支持单次游玩的分数统计
    不允许游戏运行时切换模式，避免出现逻辑错乱和用户体验问题
